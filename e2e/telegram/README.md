# Telegram E2E Assets

This directory stores Trackmate-owned Telegram E2E scenarios for the Go runtime.

Generic orchestration, MTProto login, runtime locks, binary fixtures, transcripts,
and reporting stay in the sibling `/Users/igor/projects/telegram-bot-e2e-test-tool`
repository.

## Layout

- `scenarios/` contains JSONL scenarios and topic-aware JSONL templates.
- `fixtures/` contains small Trackmate-owned text fixtures. Shared binary media
  fixtures such as `@fixtures/e2e-photo.png` are generated by the sibling tool.

## Prerequisites

Run these only against a disposable local or clearly test-only Trackmate group.
Do not point these scenarios at the production bot or production database.

The group must be a Telegram forum supergroup where the Trackmate bot is an
admin with topic-management and message-read permissions.

For a clean local Docker run:

```bash
cd /Users/igor/projects/trackmate
make docker-up
```

In another shell, prepare the sibling E2E tool:

```bash
cd /Users/igor/projects/telegram-bot-e2e-test-tool
make fixtures
make doctor
go test ./...
```

## Topic IDs

Run `scenarios/00-setup-smoke.jsonl` first on a clean test group to create or
repair Trackmate topics. Then collect the topic starter IDs with the sibling E2E
tool:

```bash
cd /Users/igor/projects/telegram-bot-e2e-test-tool
make chats CHAT_GROUPS=1 CHAT_FILTER=TrackMate CHAT_TOPICS=1 CHAT_ADMINS=1
```

Use `target=` as `TRACKMATE_CHAT`, and use the printed `topic_id=` values for:

- `TODAY_THREAD_ID` for `Сегодня`
- `PROGRESS_THREAD_ID` for `Прогресс`
- `WRONG_THREAD_ID` for any other topic in the same group, usually `Прогресс`
  when testing a pending input from `Сегодня`

## Rendering

Templates intentionally keep topic IDs as placeholders because they are different
per group.

Example render:

```bash
export TRACKMATE_CHAT='3871708263'
export TODAY_THREAD_ID='10'
export PROGRESS_THREAD_ID='11'
export WRONG_THREAD_ID="$PROGRESS_THREAD_ID"

mkdir -p tmp/e2e-rendered
for src in e2e/telegram/scenarios/*.jsonl.tmpl; do
  dst="tmp/e2e-rendered/$(basename "${src%.tmpl}")"
  sed \
    -e "s/{{CHAT}}/$TRACKMATE_CHAT/g" \
    -e "s/{{TODAY_THREAD_ID}}/$TODAY_THREAD_ID/g" \
    -e "s/{{PROGRESS_THREAD_ID}}/$PROGRESS_THREAD_ID/g" \
    -e "s/{{WRONG_THREAD_ID}}/$WRONG_THREAD_ID/g" \
    "$src" > "$dst"
done
```

Run a rendered scenario with the sibling tool:

```bash
cd /Users/igor/projects/telegram-bot-e2e-test-tool
CHAT="$TRACKMATE_CHAT" go run ./cmd/tg-e2e-tool run-scenario \
  /Users/igor/projects/trackmate/tmp/e2e-rendered/02-today-create-task.jsonl
```

`00-setup-smoke.jsonl` is not templated and can be run directly with `CHAT`.

## Scenario Catalog

- `00-setup-smoke.jsonl`: `/setup`, setup checklist refresh, and group setup
  button smoke.
- `01-today-add-prompt.jsonl.tmpl`: opens the today task prompt.
- `02-today-create-task.jsonl.tmpl`: creates one daily task in the `Сегодня`
  topic.
- `03-today-block-second-task.jsonl.tmpl`: verifies the same user cannot create
  a second daily task for the day.
- `04-report-done.jsonl.tmpl`, `05-report-partial.jsonl.tmpl`,
  `06-report-failed.jsonl.tmpl`: close a task with each report status.
- `07-wrong-topic-pending-ignored.jsonl.tmpl`: starts pending daily-task input in
  `Сегодня`, sends a decoy message in another topic, then completes the task in
  `Сегодня`.
- `08-duplicate-photo-report-consumes-once.jsonl.tmpl`: sends duplicate photo
  report messages after selecting a report status; current E2E tooling cannot
  send a true Telegram album/media group, so this is the live approximation for
  the unit-covered media-group contract.
- `09-progress-topic-event.jsonl.tmpl`: closes a task and waits for the worker to
  publish the progress event in `Прогресс`.
- `10-alert-ack.jsonl.tmpl`: acknowledges an already-dispatched daily-task alert.
  Use the local control API clock and worker tick endpoints to make alert
  dispatch deterministic before running the scenario.

## Known Execution Limits

- Topic templates must be rendered with real forum topic IDs.
- Progress and alert scenarios require the worker.
- Alert ack uses the non-production control API for clock/tick determinism.
- The sibling E2E tool can send individual media messages, but not a true
  Telegram media group/album. `08-duplicate-photo-report-consumes-once` therefore
  covers duplicate photo dispatch in live E2E, while media-group identity remains
  protected by storage and handler tests.
